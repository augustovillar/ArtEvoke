## 📋 **Dataset Information**

### **🎨 SemArt Dataset**
- **Source**: [Aston University Research Data](https://researchdata.aston.ac.uk/id/eprint/380/)
- **Images**: 21,000+ high-quality art images
- **Metadata**: Rich semantic annotations including:
  - Art type (genre, historical, landscape, portrait, etc.)
  - Detailed descriptions
  - Artist information
  - Technical details (technique, date, school)

**File Structure**:
```csv
IMAGE_FILE,DESCRIPTION,AUTHOR,TITLE,TECHNIQUE,DATE,TYPE,SCHOOL
painting1.jpg,"Medieval scene with knights","Unknown","Battle Scene","Oil on canvas","1400-1450","historical","Flemish School"
```

### **🖼️ WikiArt Dataset**
- **Source**: [Kaggle WikiArt](https://www.kaggle.com/datasets/steubk/wikiart)
- **Images**: 15,000+ images across 14 art genres
- **Organization**: Images grouped by artistic movement
- **Genres**: Baroque, Impressionism, Renaissance, Realism, etc.

**Supported Genres**:
- Baroque, Contemporary_Realism, Early_Renaissance
- High_Renaissance, Impressionism, Mannerism_Late_Renaissance
- Naive_Art_Primitivism, New_Realism, Northern_Renaissance
- Pointillism, Post_Impressionism, Realism, Romanticism, Ukiyo_e

**File Structure**:
```csv
filename,artist,genre,description
baroque/artist_artwork.jpg,"Caravaggio","Baroque","Original artwork description"
```

### **🏛️ Museum Dataset**
- **Source**: Custom collections from various museums
- **Format**: JSON with detailed metadata
- **Status**: In development

## 🔄 **Data Processing Flow**

### **Stage 1: Raw Data** (`raw/`)
- **Purpose**: Store original, unmodified datasets
- **Source**: Downloaded by `scripts/step1_data_acquisition/`
- **Format**: Original formats from data providers
- **Size**: ~50GB total

### **Stage 2: AI Descriptions** (`processed/descriptions/`)
- **Purpose**: Enhanced metadata with AI-generated descriptions
- **Generated by**: `scripts/step2_description_generation/`
- **Model**: Qwen2.5-VL-7B-Instruct
- **Enhancement**: Adds detailed visual descriptions to existing metadata

**Example Enhanced Record**:
```csv
IMAGE_FILE,TITLE,AUTHOR,ORIGINAL_DESCRIPTION,AI_DESCRIPTION
rural_scene.jpg,"Farm Landscape","Unknown","Countryside painting","A pastoral landscape showing rolling green hills with a small farmhouse nestled among trees. The scene captures a peaceful rural setting with warm earth tones and soft lighting suggesting late afternoon."
```

### **Stage 3: Vector Embeddings** (`processed/embeddings/`)
- **Purpose**: Fast semantic search capabilities
- **Generated by**: `scripts/step3_embedding_generation/`
- **Model**: thenlper/gte-large (sentence transformer)
- **Output**: FAISS indices + metadata for production use

## 📊 **File Formats & Schemas**

### **CSV Files (Metadata)**
All CSV files use UTF-8 encoding with comma separation.

**SemArt Schema**:
```python
{
    'IMAGE_FILE': str,      # Filename of the image
    'DESCRIPTION': str,     # Original description
    'AUTHOR': str,          # Artist name
    'TITLE': str,           # Artwork title
    'TECHNIQUE': str,       # Artistic technique
    'DATE': str,            # Creation period
    'TYPE': str,            # Art category
    'SCHOOL': str,          # Art school/movement
    'description': str      # AI-generated description
}
```

**WikiArt Schema**:
```python
{
    'filename': str,        # Path to image file
    'artist': str,          # Artist name
    'genre': str,           # Artistic genre
    'description': str      # AI-generated description
}
```

### **FAISS Files (Vector Search)**
- **`.faiss`**: Binary vector index for similarity search
- **`.pkl`**: Pandas DataFrame with searchable metadata
- **Embedding Model**: gte-large (1024-dimensional vectors)
- **Index Type**: IndexFlatIP (inner product similarity)

### **Image Files**
- **Formats**: JPG, JPEG, PNG
- **Resolution**: Variable (typically 512x512 to 2048x2048)
- **Color**: RGB
- **Quality**: High-resolution originals preserved

## 🔍 **Data Usage Examples**

### **Loading Processed Descriptions**
```python
import pandas as pd

# Load SemArt with AI descriptions
semart_df = pd.read_csv('processed/descriptions/output_merged_semart.csv')
print(f"SemArt dataset: {len(semart_df)} artworks")

# Filter by art type
portraits = semart_df[semart_df['TYPE'] == 'portrait']
print(f"Found {len(portraits)} portrait paintings")
```

### **Using FAISS for Search**
```python
import faiss
import pickle

# Load FAISS index and metadata
index = faiss.read_index('processed/embeddings/semart_index.faiss')
with open('processed/embeddings/semart_metadata.pkl', 'rb') as f:
    metadata = pickle.load(f)

# Search for similar artworks
# (query_vector would be generated from text using gte-large model)
distances, indices = index.search(query_vector, k=5)
results = metadata.iloc[indices[0]]
```

## 🔐 **Data Management**

### **Backup Strategy**
- **Raw Data**: Keep original downloads (can be re-downloaded)
- **Processed Data**: Backup before major pipeline changes
- **Version Control**: Use git for metadata and config files only

### **Cleanup Commands**
```bash
# Remove intermediate files
rm -rf processed/descriptions/intermediate/

# Clean and regenerate processed data
rm -rf processed/
# Re-run processing pipeline
```

### **Data Validation**
Scripts include automatic validation:
- Image file integrity checks
- CSV schema validation
- Embedding dimension verification
- Metadata consistency checks
